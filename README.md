# バイト給与計算

Googleカレンダーの予定(例:「ぐらバイト」「ミニストバイト」「塾」)を読み取って、
バイトごとの時給ルール(深夜割増・平日/休日など)に沿って給与を自動計算するWebアプリです。

## 使い方(初回セットアップ)

### 1. Google Cloud で OAuth クライアントIDを作る

1. https://console.cloud.google.com/ にアクセスし、新しいプロジェクトを作成
2. 「APIとサービス」→「ライブラリ」で **Google Calendar API** を有効化
3. 「APIとサービス」→「OAuth同意画面」を設定(External、テストユーザーに自分のGoogleアカウントを追加でOK)
4. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
   - アプリケーションの種類: **ウェブ アプリケーション**
   - 承認済みのJavaScript生成元 に、ローカルサーバーのURLを追加(例: `http://localhost:8000`)
5. 発行された「クライアントID」をコピー

### 2. クライアントIDを設定

[config.js](config.js) を開き、`YOUR_CLIENT_ID.apps.googleusercontent.com` の部分を
コピーしたクライアントIDに書き換えて保存します。

### 3. ローカルサーバーで起動

Google のログインはブラウザの `file://` では動かないため、簡易サーバーで開く必要があります。

```bash
cd baito-calc
python -m http.server 8000
```

ブラウザで `http://localhost:8000` を開いてください。
(ポート番号は手順1で設定した「承認済みのJavaScript生成元」と一致させてください)

## 使い方(日常利用)

画面下部のタブで3つの機能を切り替えます。

- **給与タブ**: 対象月(振込月)を選んで「計算する」→ その月に振り込まれる給与の合計を表示
- **設定タブ**: Googleログイン、バイトごとのキーワード・時給・締め日・支払サイクル・割増ルールを編集
  - 新しいバイトを始めたときは「+ バイトを追加」
  - 時給が変わったときは該当の時給欄を直接書き換えるだけ
- **レポートタブ**: 直近6ヶ月分の振込額をグラフと表でまとめて表示(「更新」で再取得)

## 締め日・支払サイクルの仕組み

バイトごとに以下を設定できます。

- **締め日**: 何日で勤務を区切るか(月末締めなら31を入力。実際の日数に合わせて自動調整されます)
- **支払サイクル**: 締め日を含む月を基準に「当月払い/翌月払い/翌々月払い」のいつ振り込まれるか
- **振込日**: 実際の振込日(表示用の参考情報)

例えば「月末締め・翌月払い」の場合、7/1〜7/31の勤務は8月に振り込まれる給与として集計されます。
給与タブで選んだ月には、締め日・支払サイクルが異なる複数バイトの勤務日をそれぞれ正しい月に振り分けて合算します。

## 割増ルールの仕組み

各バイトに「基本時給」と、複数の「割増ルール」を設定できます。
ルールは以下を組み合わせて指定できます。

- 曜日区分: 指定なし / 平日のみ / 休日のみ(土日+日本の祝日)
- 時間帯: 開始〜終了(日をまたぐ指定も可、例 22:00〜06:00)

勤務時間の中で条件に一致する部分だけ、そのルールの時給が適用されます
(複数条件に一致する場合は最も高い時給が採用されます)。

初期設定は以下の内容で登録済みです(締め日・支払サイクルは全て仮の値「月末締め・翌月25日払い」なので、実際の条件に合わせて設定タブで修正してください)。

| バイト | キーワード | 基本時給 | 割増ルール |
|---|---|---|---|
| ミニストバイト | ミニスト | 1160円 | 22:00〜06:00は1450円 |
| ぐらバイト | ぐら | 1150円 | 休日(土日祝)は1200円 |
| 塾 | 塾 | 1400円 | なし |

## 注意点

- 予定の「終日」イベントは集計対象外です(開始・終了時刻がある予定のみ集計)
- 予定タイトルの先頭がキーワードと一致するものだけを対象にします
- 設定はブラウザのlocalStorageに保存されます(別ブラウザ/別端末では引き継がれません)
